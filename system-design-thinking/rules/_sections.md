# Sections

This file defines all rule categories and their descriptions.

---

## 1. Foundation (foundation)

**Impact:** CRITICAL
**Description:** Rules that apply before any design work begins. Understanding requirements and constraints.

## 2. Architecture Style (style)

**Impact:** HIGH
**Description:** Rules for choosing and evolving architecture patterns. When to use monolith vs microservices.

## 3. Decomposition (decomposition)

**Impact:** HIGH
**Description:** Rules for breaking systems into components. Finding boundaries, defining contracts.

## 4. Data (data)

**Impact:** HIGH
**Description:** Rules for data architecture. Flow, ownership, storage, consistency.

## 5. Integration (integration)

**Impact:** MEDIUM
**Description:** Rules for how components communicate. Sync vs async, APIs vs events.

## 6. Operations (operations)

**Impact:** MEDIUM
**Description:** Rules for operational concerns. Deployment, monitoring, failure handling.

## 7. Documentation (documentation)

**Impact:** MEDIUM
**Description:** Rules for documenting architectural decisions. ADRs, diagrams, communication.

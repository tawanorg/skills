# JTBD Canvas Template

Populated during skill 04 (Jobs-to-be-Done). One canvas per distinct user type. The canvas is not complete until the JTBD statement has been validated with a real user.

---

## Canvas Header

```
JTBD CANVAS
─────────────────────────────────────────────────────────
Project:        ___
User type:      Primary / Secondary / Edge case
Real person:    ___  (name or anonymized ID — must be a real person observed or interviewed)
Date:           ___
Evidence grade: A / B / C / D / F
─────────────────────────────────────────────────────────
```

---

## The JTBD Statement

The core output. Must be validated with a user before locking:

```
JTBD STATEMENT
─────────────────────────────────────────────────────────
When:       ___
            (situation or trigger — specific context, not "when they want to")

I want to:  ___
            (functional job — a specific action or outcome, not a feature)

So I can:   ___
            (emotional or social outcome — what they become or how they are seen)

Without:    ___
            (primary anxiety or trade-off — what they fear losing or giving up)

─────────────────────────────────────────────────────────
VALIDATION
  Read back to user:      Done / Not done
  User response:          "That's exactly it" / "Sort of" / "Not quite"
  If not confirmed:       ___  (what needs to change)
─────────────────────────────────────────────────────────
```

## Functional Job Detail

```
FUNCTIONAL JOB
─────────────────────────────────────────────────────────
Core task:          ___  (the specific thing they are doing)
Trigger:            ___  (what causes them to start)
Frequency:          ___  (how often this trigger occurs)
Context:            ___  (where they are, what they are doing)

5 minutes before:   ___  (what they were just doing)
5 minutes after:    ___  (what they do immediately after)

Why now:            ___  (why today vs 6 months ago or 6 months from now)
Non-negotiables:    ___  (what they will not give up even for a better solution)
─────────────────────────────────────────────────────────
```

## Multi-Persona Conflict Check

If more than one persona has been defined:

```
PERSONA CONFLICT CHECK
─────────────────────────────────────────────────────────
Primary user job:   ___
Secondary user job: ___

Conflict exists:    Yes / No / Partial
Conflict point:     ___  (the specific feature or behavior where needs diverge)
Resolution:         ___  (how the conflict is resolved in the product)
Trade-off chosen:   Primary wins / Secondary wins / Both served differently
─────────────────────────────────────────────────────────
```

## Canvas Validation Status

```
CANVAS STATUS
─────────────────────────────────────────────────────────
JTBD statement validated with real user:  Yes / No
Evidence grade for functional job:        A / B / C / D / F
Evidence grade for emotional job:         A / B / C / D / F
Evidence grade for social job:            A / B / C / D / F
Adoption barriers: observed or reported:  Yes / No (assumed)

CANVAS COMPLETENESS
  [ ] JTBD statement written and validated
  [ ] Functional job documented with trigger and context
  [ ] Emotional job documented (becoming, feeling)
  [ ] Social job documented (audience, signal)
  [ ] All anxieties listed (minimum 2)
  [ ] Trade-offs listed (minimum 1)
  [ ] Non-negotiables listed (minimum 1)
  [ ] Multi-persona conflict checked if applicable

OVERALL GRADE:  Complete / Partial / UNVALIDATED
─────────────────────────────────────────────────────────
```
